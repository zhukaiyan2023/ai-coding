# MECE & SSOT

---

## 1. MECE 原则 (逻辑完备)

所有分支判断（if/switch/match）和枚举定义必须符合 MECE。

### 执行逻辑

- **不重叠 (Exclusive)**：确保路径唯一，无逻辑交叉或优先级冲突
- **不遗漏 (Exhaustive)**：严禁出现逻辑真空，必须包含 default/else/fallback 兜底

### 自检清单

- [ ] 枚举值之间无重叠
- [ ] 无 OTHER / UNKNOWN / ELSE 类型的兜底值
- [ ] switch/match 必须有 default 分支
- [ ] if-else 链必须覆盖所有可能

---

## 2. SSOT 原则 (单一真理来源)

任何事实、配置、业务逻辑在代码库中有且仅有一个权威定义。

### 执行逻辑

- **禁止硬编码**：严禁 Magic Numbers 或重复的字符串/常量
- **引用优先**：修改前必须定位原始定义，禁在调用处局部覆写

### 自检清单

- [ ] 常量是否已定义？禁止硬编码
- [ ] 业务逻辑是否已有？禁止重复实现
- [ ] 类型是否已有？禁止重复定义
- [ ] 配置是否在 config 中？禁止散落各处

---

## 3. 思维链约束 (Agent Thinking)

在输出任何方案或代码前，Agent 必须执行以下思考过程：

1. **该逻辑的 SSOT 在哪里？** 是否已存在相同逻辑？
2. **我的方案是否做到了 MECE？** 边界条件是否全部覆盖？
