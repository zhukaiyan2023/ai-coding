# Harness Engineering - 三智能体架构

## 架构图

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Planner   │────▶│  Generator  │────▶│  Evaluator │
│   (规划)    │     │   (生成)    │     │   (评估)    │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 核心原则

1. **Evaluator 独立于 Generator** - 不能自己评自己
2. **分离"做事"和"挑错"** - 独立评估更客观
3. **Sprint Contract** - 实现前协商完成标准

## 使用场景

- 复杂功能实现
- 多模块开发
- 需要严格质量把控的任务

## Sprint Contract 示例

```
Generator: "我计划实现用户注册功能，完成标准是：
  1. POST /api/users 返回 201
  2. 用户名重复返回 400
  3. 密码加密存储"

Evaluator: "同意完成标准。建议补充：
  - 邮箱格式验证
  - 密码强度要求"

双方达成一致 → 开始实现
```

## 时间分配

| 阶段 | 占比 |
|------|------|
| 规划 | 20% |
| 构建 | 40% |
| 验证 | 30% |
| 修复 | 10% |
