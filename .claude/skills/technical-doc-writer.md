# Skill: Technical Documentation Writer

## 描述
使用此 Skill 编写技术方案、架构文档、API 文档、技术决策记录等。

---

## 工作流

### Step 1: 技术方案设计

| 章节 | 内容 |
|------|------|
| 概述 | 方案简介 |
| 背景 | 问题描述 |
| 目标 | 要达成什么 |
| 非目标 | 明确不包括 |
| 方案设计 | 核心内容 |

### Step 2: 架构设计

```
┌─────────────┐
│   Client    │
└──────┬──────┘
       │
┌──────▼──────┐
│   Gateway   │
└──────┬──────┘
       │
┌──────▼──────┐
│   Service   │
└──────┬──────┘
       │
┌──────▼──────┐
│   Storage   │
└─────────────┘
```

### Step 3: 编写 API 文档

```yaml
paths:
  /resource/{id}:
    get:
      summary: 获取资源
      parameters:
        - name: id
          in: path
          required: true
      responses:
        '200':
          description: 成功
```

### Step 4: 数据字典

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| status | tinyint | 状态 0-禁用 1-启用 |

### Step 5: ADR 技术决策

```markdown
# ADR-XXX: 决策标题

## 状态
## 背景
## 决策
## 后果
```

## 检查清单

- [ ] 架构图清晰
- [ ] 接口定义完整
- [ ] 错误码覆盖全
- [ ] 数据字典准确
- [ ] 风险评估到位
- [ ] 替代方案已考虑
